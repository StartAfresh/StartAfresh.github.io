一、HTML

1.如何理解HTML5结构语义化？

HTML语义化，简单来说用正确的标签做正确的事：比如段落用p标签，标题用h1标签，侧边栏用aside

主要内容用main标签。

  • 对于开发者来说

    • 呈现较好的内容结构与代码结构，提高代码的可阅读性，

    • 便于团队的开发与维护

  • 对于浏览器

    • 有利于SEO。搜索引擎的爬虫依赖于标签来确认上下文和各个关键字的权重
    • 方便其他设备的解析，（如盲人阅读器，屏幕阅读器），利于无障碍阅读

2.html5的新特性

html5中新增了关于图像，位置，存储，多任务等功能

  • 用于媒体的video和audio元素
  • 本地存储localStorage，实现持久化存储
  • sessionStorage会话级存储
  • 语义化元素，footer、header、nav、section
  • 表单控件，date，time，email，url，search
  • 新的技术，webworker、websocket、Geolocation

移除了一些元素，比如big，font，s，u，frame

低版本浏览器（IE8、IE7、IE6）可以通过document.creatElement方法来产生标签，进行新特性的支持

4.跨域的解决办法有哪些

同源策略：Same Origin Policy SOP 是浏览器的策略

  • 同源策略(Same-Origin Policy)最早由 Netscape 公司提出，是浏览器的一种安全策略
  • 规定：只允许两个页面有相同的源时，一个页面才可以去访问另一个页面中的数据。
  • 源：说白了，就是指域名 相同的源指的是有相同的域名
  • 换句话说，jd.com不能去获取taobao.com下面的数据

有一个这样的域名：http://www.wangcai.com

  • http://zhidao.wangcai.com 不同源
  • http://www.wangcai.com:8080 不同源
  • https://www.wangcai.com 不同源
  • http://www.wangcai.com/phone/index.html 同源
  • http://www.wangcai.com/phone/huawei/index.html 同源

总结

  • 源：协议 + 域名 + 端口
  • 同源：相同的协议 && 相同域名 && 相同的端口
  • 不同源：不同的协议 || 不同的域名 || 不同的端口

不受同源策略的限制

  • 资源的引入 如：img标签的src link标签的href script标签的src
  • 页面中的超连接 a标签中的href
  • 表单的提交
  • 重定向页面

6.1 执行上下文

执行上下文就是当前 javaScript 代码被解析和执行时存在的环境（抽象概念）。

js 引擎内部有一个执行上下文栈（Execution Context Stack，简称 ECS），它是用于执行代码的调用栈。

执行上下文执行流程

JS 在执行代码时，肯定先执行全局代码，就会产生 ECG(全局执行上下文)==注意：只要有代码就有全局执行上下文==，这个 ECG 就要入栈。当我们调用一个函数，就会产生一个（局部执行上下文 EC(fn)），此时，这个局部执行上下文 EC(fn)也要入栈，当函数调用完毕后，这个 EC(fn)在堆内没有引用就要出栈（被删除）释放内存，调用其它函数又进入其他 EC

执行完毕后又，出以此类推,当全局代码执行完后，EC(G)也要出栈。

3.什么是虚拟DOM?

虚拟dom本质上就是一个普通的Js对象,(使用JS对象模拟真实DOM)一个真实DOM的中的属性非常多，操作真实DOM性能就差一点，采用JS对象模拟真实DOM它的属性就可以少很多，浏览器运行JS的速度肯定比操作DOM的性能要高

4.说说你对Vue生命周期的理解

组件从创建到销毁的过程就是它的生命周期

创建

beforeCreat

  • 生命周期，事件，数据代理还未开始，这个阶段vm中的date和方法都无法使用

Created

  • 初始化响应式系统（data中的数据和methods可以访问了）

挂载

beforeMount

  • 完成了模板的编译，内部调用render函数生成虚拟DOM，即将渲染
  • 内部调用render函数生成虚拟DOM（使用JS对象模拟真实DOM）
  • 一个真实DOM的中的属性非常多，操作真实DOM性能就差一点，如何采用JS对象模拟真实DOM
    它的属性就可以少很多，浏览器运行JS的速度肯定比操作DOM的性能。
    虚拟DOM要转人成真实DOM，进行挂载，只有挂载到DOM树上，页面才会显示

mounted

  • 把编译好的模板挂载到页面，这里可以发送异步请求，可以访问DOM节点

更新

beforeUpdate

  • 组件数据更新之前使用，数据是新的，页面上的数据是旧的。 要生成新的虚拟DOM，新的虚拟DOM和老的虚拟DOM进行对比，找到差异会执行diff算法，找到两个虚拟DOM的最小差异，找到后，进行异步更新

updated

  • render 重新做了渲染，这时数据和页面都是新的（避免在这里更新数据，会导致死循环 ）

销毁

beforeDestroyed

  • 在这里vm还可以用。
  • 在这里一般做一些收尾工作
    1）关闭定时器 2）DOM事件的解绑 3）优化项目性能 4）清缓存…

destroyed

  • 组件销毁了

添加deep：true这个属性，实现 深度监听，但是监听的数据不能获得oldValue，因为新的value和旧的value指向的是同一个对对象

6.删除数组用delete和Vue.delete有什么区别？

delete是将删除的元素改成empty/undefined，元素的值和长度不会发生改变。

Vue.delete是直接删除数组的元素，键值和数组长度会改变

7.watch和computed有什么区别？

  1. computed: 通过已有的属性，计算出新的属性，底层借助了 Objcet.defineproperty（）方法提供的 getter 和 setter
    • 初次读取时会执行一次
    • 当依赖的数据发生改变时会再次执行
    • 内部有缓存机制（复用）
    • 计算的属性最终会出现在vm身上
  2. watch:
    • 当被监视的属性变化时，回调函数自动调用，可以监听的数据来源：data，props，computed内的数据；
    • 深度侦听：在 watch 中配置 deep:true 可以监测对象内部值的改变（多层）
  3. 计算属性 VS 侦听属性
    • 异步操作
    • 缓存操作

8.v-for没有key会发生什么问题？

  • key是虚拟 dom 中对象的标识，当数据发生改变时，vue会根据新数据生成新的虚拟DOM随后进行新虚拟DOM与旧虚拟DOM进行差异比较。
  • 对比规则
    1. 旧虚拟 dom 中找到了与新虚拟 dom 相同的 key
      1. 若虚拟 dom 中内容没变直接使用之前的真实 dom
      2. 若虚拟 DOM 中内容变了，则生成新的真实 DOM,随后替换掉页面中之前的真实 DOM
    2. 旧虚拟 dom 中未找到与新虚拟 dom 相同的 key 创建新的真实 DOM，随后渲染到到页面
  • index作为key可能引发的问题
    1. 若对数据进行逆序添加，逆序删除等破坏顺序的操作，会产生没有必要的真实 D0M 更新==>界面效果没问题，但效率低
    2. 若结构中还包含输入类的 D0M：会产生错误 D0M 更新==>界面有问题

3.在created和mounted去请求数据，有什么区别？

created:在渲染前调用，通常先初始化属性，然后做渲染
mounted:在模板渲染完成后，一般都是初始化页面后，在对元素节点进行操作
在这里请求数据可能会出现闪屏的问题，created里不会
一般用created比较多
请求的数据对DOM有影响，那么使用created
如果请求的数据对DoM无关，可以放在mounted
